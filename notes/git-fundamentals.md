Git Fundamentals
What is Git?

Git is a distributed version control system designed to track changes in source code during software development.

It allows developers to:

Track changes

Revert to previous versions

Collaborate efficiently

Maintain a complete project history

Repository

A repository (repo) is a project tracked by Git.

It contains:

Project files

Change history

Configuration data

Repositories can be:

Local (on your machine)

Remote (on platforms like GitHub)

Commit

A commit is a snapshot of the project at a specific point in time.

Each commit:

Has a unique ID (hash)

Includes a message

Records changes made since the last commit

Commits build the project history.

Branch

A branch is an independent line of development.

The default branch is usually called:

main

Branches allow:

Feature development

Testing

Safe experimentation

Without affecting the main codebase.

Remote

A remote repository is a version of your project hosted online.

Common workflow:

Make changes locally

Commit changes

Push to remote repository

This ensures backup, collaboration and portfolio visibility.
